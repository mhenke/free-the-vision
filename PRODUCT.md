# Product

## Register

brand

## Users

Broader tech public — developers, PMs, tech leads, and anyone who uses or evaluates GitHub Copilot. The audience knows what Copilot is; they don't need explaining. They're here because they want Vision to be GA, and they want to laugh a little while demanding it.

## Product Purpose

A campaign page rallying support for making GitHub Copilot Vision (image upload) Generally Available. The site makes the case through the lens of a Free Willy parallel: Vision is trapped in preview, and the community wants it freed. Success means the page is shared, the hashtag trends, and GitHub notices.

## Brand Personality

Tongue-in-cheek — serious cause, playful delivery. The Free Willy parallel is played with a wink, not full parody. The whale metaphor has warmth and humor. Voice is direct, developer-native, no marketing fluff. Think "the GitHub community wrote a really good blog post that also happens to be a campaign."

## Anti-references

AI slop landing page: cream background, gradient CTAs, numbered `01 / 02 / 03` section markers, tiny uppercase tracked eyebrows above every section, identical card grids, frosted glass cards, generic "modern" typography, hero-metric templates.

Also not: corporate GitHub blog post (blue, neutral, dry), template marketing pages (any site built from a Figma template).

## Design Principles

1. **Campaign page, not product page.** The design IS the product. One dominant idea per fold. Long scroll with deliberate pacing. No card grids.
2. **Earnest message, playful delivery.** The whale metaphor is played with warmth and humor. The cause is real; the tone doesn't take itself too seriously.
3. **Image-led.** Ocean photography carries the visual weight. Real imagery, not CSS scenery or decorative diagrams.
4. **Developer-native.** Honest, direct copy. No marketing superlatives. The audience smells BS instantly.
5. **One signature moment.** The hero entrance is the page's signature. Everything else is quieter.

## Accessibility & Inclusion

- WCAG AA compliance
- Reduced motion: all animations have `prefers-reduced-motion` fallback
- Keyboard navigable: all interactive elements reachable and operable
- Semantic HTML: proper landmarks, headings, labels
- Alt text on all imagery: descriptive, voice-aligned
- Contrast: body text ≥ 4.5:1, large text ≥ 3:1
